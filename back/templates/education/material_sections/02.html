Установка фреймворка

Pip

Начиная с версии Python 3.4, в составе исполняющей среды этого языка поставляется утилита pip, с помощью которой очень удобно выполнять установку любых дополнительных библиотек. Эта утилита самостоятельно ищет указанную при ее запуске библиотеку в штатном репозитории PyPI (Python Package Index, реестр пакетов Python) - интернет-хранилище самых разных библиотек для Python. Найдя запрошенную библиотеку, pip самостоятельно загружает и устанавливает наиболее подходящую ее редакцию, при этом загружая и устанавливая также и все библиотеки, которые она использует для работы.

Виртуальные среды

Виртуальные среды - неотъемлемая часть программирования на Python. Они представляют собой изолированный контейнер, содержащий все зависимости программного обеспечения для данного проекта. Этот важно, потому что по умолчанию разные библиотеки, устанавливается в одном каталоге. Это вызывает проблему, если вы хотите работать над несколькими проектами на одном компьютере. Что, если Проект_А использует Django 3.2, но Проект_Б с прошлого года все еще находится на Django 2.1? Без виртуальных сред это становится очень сложно.

Есть много областей разработки программного обеспечения, о которых постоянно ведутся споры, но использование виртуальных сред для разработки на Python - не одна из них. Вы должны использовать новую виртуальную среду для каждого нового проекта Python.

Исторически разработчики Python использовали virtualenv или pyenv для настройки виртуальных сред. Но в 2017 году известный разработчик Python Кеннет Рейц выпустил Pipenv, который теперь является официально рекомендованным инструментом упаковки Python.

Первый способ установки

Давайте установим pipenv 

Command Line
        pip install pipenv

Теперь создадим папку с нашим проектом, пусть будет DjangoProject 

Command Line
        mkdir DjangoProject

cd DjangoProject

Теперь установим Djаngo используя pipenv

Command Line
        pipenv install django

Если вы заглянете в наш каталог, то увидите два новых файла: Pipfile и Pipfile.lock. У нас есть информация, необходимая для новой виртуальной среды, но мы еще не активировали ее. Давайте сделаем это с помощью pipenv. 

Command Line
        pipenv shell

Если вы работаете на Mac, теперь вы должны увидеть круглые скобки в командной строке с активированной средой. Он будет иметь формат имени каталога и случайных символов. В моем случае это DjangoProject-JmZ1NTQw 

Command Line
        (DjangoProject-JmZ1NTQw) $

Значит, работает!
